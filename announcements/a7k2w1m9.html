核心挑戰：24GB VRAM 的物理天花板

在 24GB 顯示卡（如 RTX 3090/4090 或 A5000）上部署本地 Agent 時，最大的誤區是「盲目追求參數大小」。過度壓縮（量化）大模型反而會導致邏輯像素遺失，讓模型變笨。

三大避坑指南

1. MTP (Multi-Token Prediction) 的陷阱

MTP 雖然能提升生成速度，但在 Agent 任務中，若第一步推理偏差，MTP 會像骨牌效應一樣放大錯誤。對於需要嚴謹邏輯的 Agent 迴圈，MTP 往往是毒藥。

2. 注意力稀釋與 KV Cache

Agent 在長時間對話後會「崩潰」（如重複說話或遺忘指令），這是因為 KV Cache 堆疊導致注意力被稀釋。務必進行 20 輪對話壓力測試，觀察模型在長時間工作後的穩定性。

3. 稠密 (Dense) vs 混合專家 (MoE)

在 24GB 顯卡上，MoE 模型（如 40B 總參數但僅 12B 活躍）通常能展現出遠超預期的生成速度，且保有強大知識量，是極具性價比的選擇。

實戰起手式總結

  • 基準線 (Baseline)： 從 Llama 3.1 8B Instruct 開始，測試基本反應。
  • 進階規劃： 需要多步驟推理時，首選 Qwen 2.5 14B Instruct。
  • 知識型助理： 追求真實世界常識與流暢體感，選擇 Gemma 2 9B Instruct。
  • 驗證方法： 執行「Side-by-Side」並排測試，固定 20 輪對話壓力。
